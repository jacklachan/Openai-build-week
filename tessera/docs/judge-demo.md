# Tessera - three-minute judge run

This is the recording and live-demo path. It works with no API key, payment method, or connected messaging account.

## 0:00-0:20 - The thesis

Say: "Group travel tools make an itinerary. Tessera makes the compromise inspectable, so the quiet person does not lose their must-do."

On the landing screen, click **Try the Japan chat** or import a WhatsApp `.txt` export. Point out that chat signals become editable preferences and the raw export is not being sent by this local intake step.

## 0:20-0:55 - The one decision

Click **Load the Japan demo**, then **Ask the one question**. Read Priya's 05:30-start question aloud and point to the Tokyo -> Fuji/Hakone -> Kyoto -> Osaka route.

Choose **Yes - protect Ravi's sunrise**. Show the decision receipt: what is protected, what shifted in the route, and the budget effect. Say: "The model is not allowed to hide a compromise behind an itinerary."

## 0:55-1:35 - The agreement

Open the pact. Point to each traveler's named win and written concession. Use **Flag a concern**, preview the veto, and apply it. Show the before/after audit.

Say: "Every change has a receipt: the activity removed, activity added, budget delta, and updated trade-off."

## 1:35-2:05 - The resilience test

In Proposal Arena select **Pressure-test this pact**. Explain that it is deliberately labelled a simulated disruption, not a weather feed. Show who carries the cost if the exposed stop changes, then select the visible alternative.

Say: "Tessera is useful when reality changes, not only when everyone is still excited in the group chat."

## 2:05-2:30 - Back to the group

Complete the check-in, then use **Share with group** or **WhatsApp**. Explain that the browser opens a user-controlled handoff: Tessera does not ask for WhatsApp credentials or operate as a hidden bot.

## 2:30-3:00 - Build evidence

Say: "GPT-5.6 through Codex built the product architecture, contract validation, 3D route experience, decision replay, disruption drill, and test suite. The optional Gemini provider produces JSON only; Tessera validates it and recomputes the budget. The no-key path stays deterministic so judges can run it exactly."

Show the README verification commands and this repository's commit history briefly. Add the primary Codex `/feedback` session ID to `docs/build-week-submission.md` and the Devpost form before submitting.
